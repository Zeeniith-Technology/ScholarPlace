## Part 1.1: Why Percentages Were Invented 

### The Problem Before Percentages

**Ancient Trade Scenario (1000 CE):**
```
Merchant A sells cloth: 15 out of 30 pieces sold
Merchant B sells silk: 25 out of 50 pieces sold
Who is better at selling?

Without percentages: 
- A sold 15 pieces (smaller number)
- B sold 25 pieces (larger number)
- Conclusion: B is better? (25 > 15)

With percentages:
- A: (15/30) √ó 100 = 50% sold
- B: (25/50) √ó 100 = 50% sold
- Conclusion: Both equally good!

Percentages allow fair comparison across different-sized groups.
```

**Why it matters today:**
- Compare companies of different sizes
- Understand growth rates fairly
- Make informed decisions
- Avoid being misled by raw numbers

---

## Part 1.2: What is a Percentage? 

### Formal Mathematical Definition

**Definition 1 (Standard):**
A percentage is a number expressed as a fraction of 100. It represents a ratio comparing a part to a whole on a standardized scale of 100.

**Notation:**
```
Percentage = (Part / Whole) √ó 100
Symbol: %
```

**Definition 2 (Conceptual):**
Percentage is a PROPORTIONAL COMPARISON - it shows what amount you'd have if the total were exactly 100.

**Definition 3 (Practical):**
Percentage tells you "for every 100 units of reference, how many units of measurement?"

### Why BASE (Whole) is CRITICAL

**Key Insight:** The "whole" (base) is YOUR REFERENCE POINT. It's the thing everything else is compared to.

**Deep Concept Example 1A: Pass Rates**

School A: 15 students passed, 30 total students
School B: 20 students passed, 50 total students

**WITHOUT understanding base:**
- School B had more passes (20 > 15) ‚Üí Seems better
- But this is MEANINGLESS comparison (different group sizes!)

**WITH understanding base:**
- School A: 15/30 = 50% pass rate (BASE = 30 total)
- School B: 20/50 = 40% pass rate (BASE = 50 total)
- Conclusion: School A is BETTER despite fewer actual passes!

**The insight:** Same absolute number (passes) but DIFFERENT relative importance because different bases.

This is why percentage is so powerful - it removes base distortion.

## Part 1.3: The Percentage Formula 

### Basic Formula Explained

```
Percentage = (Part / Whole) √ó 100
```

**Each component:**
- **Part:** What we're measuring / What we're interested in
- **Whole:** Total amount / Reference point / Base
- **√ó 100:** Scale to 100 (standardization)

### Why Multiply by 100?

**Without √ó100:**
```
15/30 = 0.5 (this is hard to compare)
```

**With √ó100:**
```
15/30 √ó 100 = 50 (much easier - out of 100 standard)
```

Multiplying by 100 converts to standard scale where comparison is intuitive.

## Part 1.4: Understanding Percentage Change 

### What "Change" Really Means

**Common mistake:**
"Value changed from 20 to 25" 
Student thinks: "5 out of 25 = 20%"

**Reality:**
"Value changed from 20 to 25"
Correct: "5 out of 20 (original) = 25%"

**WHY?** Change is always calculated from the ORIGINAL value, never the new value.

### The Full Percentage Change Formula

```
Percentage Change = (Change / Original) √ó 100

Where:
Change = New Value - Original Value
Original = Starting Point (this is the BASE)
```

### Deep Example 1A: Salary Increase 

**Scenario:**
Employee A: Salary ‚Çπ20,000 ‚Üí ‚Çπ24,000
Employee B: Salary ‚Çπ1,00,000 ‚Üí ‚Çπ1,02,000

**Which employee got a better raise?**

**Calculation for A:**
```
Change = 24,000 - 20,000 = ‚Çπ4,000
Original = ‚Çπ20,000
% Change = (4,000 / 20,000) √ó 100 = 20%
```

**Calculation for B:**
```
Change = 1,02,000 - 1,00,000 = ‚Çπ2,000
Original = ‚Çπ1,00,000  
% Change = (2,000 / 1,00,000) √ó 100 = 2%
```

**The Insight:**
- A got 20% raise (better!)
- B got 2% raise
- Even though B got more absolute money (‚Çπ2,000 vs ‚Çπ4,000), A's percentage is higher

**Why?** A's base (‚Çπ20K) is smaller, so the same ‚Çπ4K is a bigger proportion of it.

**Life Application:**
If you earn ‚Çπ20K and get ‚Çπ4K raise, that's significant (20%).
If you earn ‚Çπ100K and get ‚Çπ4K raise, that's negligible (4%).

---

### Deep Example 1B: Investment Returns

**Scenario:**
Investor X: Invested ‚Çπ100, earned ‚Çπ50 (50% return)
Investor Y: Invested ‚Çπ1,000, earned ‚Çπ500 (50% return)

**Question:** Who made more money?

**Answer:** Both earned 50% but...
- X has ‚Çπ150 total (doubled + extra)
- Y has ‚Çπ1,500 total (doubled + extra)

Y is much richer even though both got 50% return!

**Lesson:** Percentage and absolute amount are both important. Same percentage with different bases = different outcomes.

### The Multiplier Method (Alternative Representation)

**Instead of thinking about formulas, think about multipliers.**

**If something increases by x%:**
```
New Value = Original √ó (1 + x/100)
Multiplier = 1 + x/100
```

**If something decreases by x%:**
```
New Value = Original √ó (1 - x/100)
Multiplier = 1 - x/100
```

**Examples:**

Increase by 20%:
```
Multiplier = 1 + 0.20 = 1.20
If original = 100:
New = 100 √ó 1.20 = 120 ‚úì
```

Decrease by 30%:
```
Multiplier = 1 - 0.30 = 0.70
If original = 100:
New = 100 √ó 0.70 = 70 ‚úì
```

**Why multipliers matter:** Makes successive changes MUCH easier.

## Part 1.5: Increase vs Decrease (Asymmetry Discovery)

### Increasing Percentage

**Formula:**
```
New = Original √ó (1 + rate/100)
Increase% = (New - Original) / Original √ó 100
```

**Deep Example 2A: Store Inflation**

```
Original price: ‚Çπ500 (for a product)
Inflation rate: 12% increase
Calculation:
New price = 500 √ó (1 + 12/100) = 500 √ó 1.12 = ‚Çπ560

Verification:
Increase = 560 - 500 = ‚Çπ60
Rate = (60/500) √ó 100 = 12% ‚úì

Meaning: For every ‚Çπ100 of original price, you now pay ‚Çπ112
Or: For every ‚Çπ100, you pay ‚Çπ12 extra
```

---

### Decreasing Percentage

**Formula:**
```
New = Original √ó (1 - rate/100)
Decrease% = (Original - New) / Original √ó 100
```

**Deep Example 2B: Store Discount**

```
Marked price: ‚Çπ1,000
Discount: 25%
Calculation:
Sale price = 1,000 √ó (1 - 25/100) = 1,000 √ó 0.75 = ‚Çπ750

Verification:
Decrease = 1,000 - 750 = ‚Çπ250
Rate = (250/1,000) √ó 100 = 25% ‚úì

Meaning: For every ‚Çπ100 of marked price, you pay ‚Çπ75
Or: You save ‚Çπ25 on every ‚Çπ100
```

### CRITICAL INSIGHT: +x% Then -x% ‚â† Back to Original

**This is the most important discovery of Day 1.**

**The Question:**
If a price increases by 20% then decreases by 20%, do you end up where you started?

**Intuitive answer:** Yes (symmetric changes)

**Actual answer:** NO - you end up LOWER!

**Proof:**

```
Start: ‚Çπ100

Step 1: Increase by 20%
Multiplier = 1.20
New = 100 √ó 1.20 = ‚Çπ120

Step 2: Decrease by 20% (from ‚Çπ120, not ‚Çπ100!)
Multiplier = 0.80
Final = 120 √ó 0.80 = ‚Çπ96

Result: ‚Çπ96 ‚â† ‚Çπ100 (We're DOWN ‚Çπ4!)
```

**Why This Happens:**

- +20% adds: 100 √ó 0.20 = ‚Çπ20 (20% of ‚Çπ100)
- -20% subtracts: 120 √ó 0.20 = ‚Çπ24 (20% of ‚Çπ120, which is larger!)

‚Çπ24 > ‚Çπ20, so net loss!

**General Formula (CRITICAL FOR INTERVIEWS):**

```
If you do +x% then -x%:

Net multiplier = (1 + x/100) √ó (1 - x/100)
                = 1 - (x¬≤/10000)
                = 1 - x¬≤/100 % loss

NET LOSS = (x¬≤/100) %
```

**Examples:**

+10% then -10%:
```
Loss = (10¬≤/100) = 100/100 = 1% loss
Final = 99% of original
```

+20% then -20%:
```
Loss = (20¬≤/100) = 400/100 = 4% loss
Final = 96% of original
```

+50% then -50%:
```
Loss = (50¬≤/100) = 2500/100 = 25% loss
Final = 75% of original
```

**Key Learning:** Increases and decreases are NOT symmetric! Decreases hurt more.

## Part 1.7: Common Mistakes

### Mistake 1: Using New Value as Base

**Student Error:**
```
"Salary increased from ‚Çπ20,000 to ‚Çπ24,000"
Student calculates: 4,000 / 24,000 √ó 100 = 16.67%
```

**Why It's Wrong:**
Percentage change ALWAYS uses ORIGINAL as base, never final value!

**Correct:**
```
4,000 / 20,000 √ó 100 = 20%
```

**Cognitive Reason:**
Students focus on the latest/most visible number. This is called "recency bias" - brain defaults to most recent information.

**Prevention:**
Always ask: "Compared to what?"
If problem says "increase," it's compared to ORIGINAL.

---

### Mistake 2: Assuming Percentage is Always Out of 100

**Student Error:**
```
"75% of 200 is 75"
Student thinks: Percentage = actual part
```

**Why It's Wrong:**
Percentage is a RATIO, not an absolute amount.

**Correct:**
```
75% of 200 = (75/100) √ó 200 = 150
```

**Prevention:**
Remember: % shows RELATIONSHIP, not absolute value.
200 might be 100%, 75 might be 75%.

---

### Mistake 3: Thinking +10% then -10% = 0%

**Student Error:**
```
"Stock up 10%, then down 10%. Back to original!"
```

**Why It's Wrong:**
Decreases are on larger base, so subtract more.

**Correct:**
```
+10%: Multiply by 1.10
-10%: Multiply by 0.90 (on the new amount)
Result: 1.10 √ó 0.90 = 0.99 = 99% (1% loss!)
```

**Cognitive Reason:**
Symmetry bias - students expect symmetric operations to cancel out. They don't in percentages!

## Part 2.1: Why Successive Percentages Are Different

### The Core Problem: Compounding

**Real-Life Scenario:**
```
Company growth over 2 years:
Year 1: 30% growth
Year 2: 20% growth

CEO thinks: Total growth = 30% + 20% = 50%

But is this correct?
```

**Analysis:**

```
Start: ‚Çπ100 (company value)
After Year 1 (30% growth): ‚Çπ100 √ó 1.30 = ‚Çπ130
After Year 2 (20% growth): ‚Çπ130 √ó 1.20 = ‚Çπ156

Total growth: (156 - 100) / 100 √ó 100 = 56%

NOT 50%!
```

**Why:** Year 2's 20% is calculated on ‚Çπ130 (larger base), not original ‚Çπ100.

---

## Part 2.2: The Successive Percentage Formula 

### Formula:

```
Net% = a + b + (a √ó b / 100)

Where:
a = first percentage (+ for increase, - for decrease)
b = second percentage (+ for increase, - for decrease)
```

### Mathematical Proof:

```
Let Original = P

After first change: P √ó (1 + a/100)
After second change: P √ó (1 + a/100) √ó (1 + b/100)

Expand the product:
= P √ó [1 + a/100 + b/100 + (a√ób)/10000]
= P √ó [1 + (a + b + ab/100)/100]

So the net percentage = a + b + (ab/100)
```

### Visual Understanding:

```
Change 1: a% of P = (a/100) √ó P
Change 2: b% of (P + Change 1) = (b/100) √ó (P + Change 1)
          = (b/100) √ó P + (b/100) √ó (a/100) √ó P
          = (b/100) √ó P + (ab/10000) √ó P

Total change = (a/100) √ó P + (b/100) √ó P + (ab/10000) √ó P
             = [(a + b + ab/100)/100] √ó P
             = a + b + (ab/100) as percentage

The (ab/100) term is the "interaction" - it's how much the second change affects the extra created by first change.
```

## Part 2.4: Profit, Loss & Discount Applications

### Part 2.4A: Profit and Loss 

**Definitions:**

```
Cost Price (CP) = What we paid to acquire
Selling Price (SP) = What we sold it for
Profit = SP - CP (if positive)
Loss = CP - SP (if positive)

Profit% = (Profit / CP) √ó 100
Loss% = (Loss / CP) √ó 100

KEY: Base is ALWAYS CP, because profit is on your investment!
```

**Why CP is Base:**

```
If you invested ‚Çπ400 and sold for ‚Çπ520:
Profit = ‚Çπ120
Profit% = (120/400) √ó 100 = 30%

This means: "For every ‚Çπ100 you invest, you earn ‚Çπ30 profit"

If we used SP as base: (120/520) √ó 100 = 23% - WRONG, doesn't show return on investment
```

---

### Deep Example 2A: Shopkeeper Profit

```
Q: A shopkeeper buys a product for ‚Çπ400 and sells for ‚Çπ520.
   Find profit and profit percentage.

Solution:
CP = ‚Çπ400
SP = ‚Çπ520
Profit = 520 - 400 = ‚Çπ120

Profit% = (120/400) √ó 100 = 30%

Interpretation:
- For every ‚Çπ100 of cost price, profit is ‚Çπ30
- For every ‚Çπ100, selling price is ‚Çπ130

If buying goods worth ‚Çπ10,000:
Expected profit = 30% of 10,000 = ‚Çπ3,000
Selling price should be = ‚Çπ13,000
```

### Part 2.4B: Discount

```

Discount% = (Discount / MP) √ó 100

KEY: Base is ALWAYS MP, because discount is reduction from marked price!
```

---

### Deep Example 2B: Store Discount

```

   What does customer pay?
```

### Part 2.4C: Multiple Discounts

**The Mistake:** Adding discounts

```

Correct: 20% then 10% is NOT 30%!
```

```

First discount 20%:
Amount reduced = 1,000 √ó 0.20 = ‚Çπ200
Price becomes = 1,000 - 200 = ‚Çπ800

Second discount 10% (on ‚Çπ800, NOT original ‚Çπ1,000):
Amount reduced = 800 √ó 0.10 = ‚Çπ80
Price becomes = 800 - 80 = ‚Çπ720

Total reduction: (1,000 - 720) / 1,000 √ó 100 = 28%

NOT 30%!
```

**Using Formula:**

For successive discounts, use:
```
Net discount% = a + b - (a √ó b / 100)

NOTE: SUBTRACT the product, not add (opposite of growth)
```

```
= 20 + 10 - (20 √ó 10 / 100)
= 30 - 2
= 28% ‚úì
```

# üìÖ DAY 3: RATIO & PROPORTION 

## Part 3.1: What is a Ratio? 

### Misconception to Eliminate First

**Wrong idea:** "Ratio 3:5 means the value is 3 or 5 or 3/5"

**Correct idea:** "Ratio 3:5 is a RELATIONSHIP describing how two quantities compare"

---

### Formal Definition

**Definition 1 (Mathematical):**
Ratio is a comparison of two quantities using division, expressed in the form a:b, where a and b are numbers.

**Definition 2 (Conceptual):**
Ratio tells us "for every a units of one thing, there are b units of another thing."

**Definition 3 (Proportional):**
Ratio is a fixed relationship that remains the same when both quantities are scaled by the same factor.

---

### Deep Example 3A: School Class

```
Teachers : Classrooms = 2 : 3

Interpretation:
"For every 2 teachers, there are 3 classrooms"

This could mean:
- 2 teachers, 3 classrooms (actual size)
- 4 teachers, 6 classrooms (doubled)
- 20 teachers, 30 classrooms (multiplied by 10)
- 1.5 teachers, 2.25 classrooms (fractional scaling)

All have same ratio 2:3!
```

---

### Why Ratio Matters: The Scaling Power

```
School A: 2 teachers, 3 classrooms (ratio 2:3)
School B: 20 teachers, 30 classrooms (ratio 2:3)
School C: 10 teachers, 15 classrooms (ratio 2:3)

All have SAME ratio, but different sizes!

With ratios, we can:
1. Compare despite different sizes
2. Scale up or down
3. Predict proportions
```

---

## Part 3.2: Equivalent Ratios (Scaling Rules)

### Core Principle

**If you multiply (or divide) BOTH parts of a ratio by the same number, the ratio stays the same.**

```
2:3 = 4:6 = 6:9 = 10:15 = 100:150

All are equivalent ratios
```

### Simplification (Reducing to Simplest Form)

```
100:150
Divide both by 10: 10:15
Divide both by 5: 2:3 (simplest form)

So 100:150 = 2:3 in simplest form
```

**Why simplify?** Makes it easier to understand.

---

### Deep Example 3B: Recipe Scaling

```
Original recipe (serves 2 people):
Flour: 200g
Sugar: 300g
Ratio flour:sugar = 200:300 = 2:3

Now scale for 10 people (5x larger):
Flour: 200 √ó 5 = 1,000g
Sugar: 300 √ó 5 = 1,500g
Ratio: 1,000:1,500 = 2:3 ‚úì (still same!)

This is why recipes use ratios - they scale perfectly!
```

---

## Part 3.3: Proportion (Equal Ratios with Formula)

### Definition

**Proportion:** Statement that two ratios are equal

```
Notation: a:b :: c:d (read as "a is to b as c is to d")
Or: a/b = c/d
```

---

### The Cross Multiplication Rule

**If a:b = c:d, then a √ó d = b √ó c**

**Proof:**

```
If a/b = c/d
Multiply both sides by b √ó d:
(a/b) √ó (b √ó d) = (c/d) √ó (b √ó d)
a √ó d = b √ó c
```

---

### Deep Example 3C: Finding Missing Value

```
Q: If 3:4 = 12:x, find x

Solution using cross multiplication:
3 √ó x = 4 √ó 12
3x = 48
x = 16

Check: 3:4 = 0.75 and 12:16 = 0.75 ‚úì
```

---

### Deep Example 3D: Recipe Scaling Precision

```
Q: Recipe for 2 people needs 200g flour, 300g sugar
   How much flour and sugar for 5 people?

Solution using proportion:
People‚ÇÅ/Flour‚ÇÅ = People‚ÇÇ/Flour‚ÇÇ
2/200 = 5/F
2F = 200 √ó 5
2F = 1,000
F = 500g flour

Similarly for sugar:
2/300 = 5/S
S = 750g sugar

Alternative (using ratio):
Ratio = 200:300 = 2:3
For 5 people: 2.5 √ó (2:3) = 5:7.5
So 5 parts flour units and 7.5 parts sugar units
If 1 unit = 100g: flour = 500g, sugar = 750g ‚úì
```

## Part 3.4: Types of Proportions

### Type 1: Direct Proportion (y increases when x increases)

**Formula:** y = kx (where k is constant)

**Real Example:**

```

```

**Formula:** y = k/x (or xy = k)

```
- 1 worker: 100 days
- 2 workers: 50 days
- 5 workers: 20 days
- 10 workers: 10 days
```

## Part 3.5: Compound Ratios (Three or More Ratios)

### The Challenge

```
A:B = 3:4
B:C = 2:5

Find A:C = ?

Problem: B has different values (4 in first ratio, 2 in second)
Solution: Make B equal in both ratios
```

### Solution Process

```
Step 1: Find LCM of B values (4 and 2)
LCM(4, 2) = 4

Step 2: Scale first ratio so B = 4
A:B = 3:4 ‚úì (already 4)

Step 3: Scale second ratio so B = 4
B:C = 2:5
Multiply both by 2: B:C = 4:10

Step 4: Combine
A:B:C = 3:4:10

Therefore A:C = 3:10
```

## Part 4.1: Direct Proportion

### Definition and Formula

**Definition:**
Two quantities are in direct proportion if one increases exactly as the other increases by same factor.

**Formula:** y = kx (where k is constant of proportionality)

**Also:** y/x = k (constant ratio)

---

### Why It Matters

```
Real Examples:
1. Total cost = Price √ó Quantity
2. Distance = Speed √ó Time
3. Salary = Daily wage √ó Days worked
4. Area = Length √ó Width

All follow y = kx pattern
```

---

### Deep Example 4A: Constant Speed Travel

```

Solution:
Time (hours) | Distance (km)
1            | 60
2            | 120
3            | 180
4            | 240

Pattern: Distance = 60 √ó Time
Here k = 60 (speed)

Straight line through origin when plotted!
```

---

### Property of Direct Proportion

```
If x‚ÇÅ:x‚ÇÇ = y‚ÇÅ:y‚ÇÇ (ratio of x equals ratio of y)

Example:
If x doubles, y doubles
If x triples, y triples
```

---

## Part 4.2: Inverse Proportion

### Definition and Formula

**Definition:**
Two quantities are in inverse proportion if one increases as the other decreases, such that their product remains constant.

**Formula:** xy = k (or y = k/x)

**Also:** x‚ÇÅy‚ÇÅ = x‚ÇÇy‚ÇÇ (products equal)

---

### Deep Example 4B: Work Completion

```
Q: 100 worker-days of work needed.
   Make table for workers vs days

4       | 25   | 100
5       | 20   | 100
10      | 10   | 100
20      | 5    | 100

Pattern: Workers √ó Days = 100 (constant)

Hyperbola shape when plotted!
```

---

### Property of Inverse Proportion

```
If x‚ÇÅ:x‚ÇÇ = y‚ÇÇ:y‚ÇÅ (ratio of x equals INVERSE ratio of y)

Example:
If workers double, days halve
If workers triple, days become 1/3
```

---

## Part 4.3: Identifying Proportion Type

### Decision Framework

```
Q: Is this direct or inverse?

Questions to ask:
1. As one increases, does other increase? ‚Üí DIRECT
2. As one increases, does other decrease? ‚Üí INVERSE
3. Is the ratio constant (y/x = k)? ‚Üí DIRECT
4. Is the product constant (x√óy = k)? ‚Üí INVERSE
```

---

### Real Examples to Identify

```
1. Price per item √ó Number of items = Total cost
   ‚Üí DIRECT (both increase together)

2. Speed √ó Time = Distance
   ‚Üí DIRECT (as time increases, distance increases)

3. Workers √ó Days = Total work
   ‚Üí INVERSE (more workers means fewer days)

4. Weight of object √ó Distance from fulcrum = Torque (moment)
   ‚Üí DIRECT (both increase together)

5. Concentration √ó Volume = Amount of substance
   ‚Üí DIRECT (both increase together)
```

## Part 5.2: Population and Growth Problems

### Deep Example 5C: Population Growth

```

Step 3: Calculate CAGR (Compound Annual Growth Rate)

This is for 3 years.
Annual rate = (1.264032)^(1/3) = 1.0814
So CAGR ‚âà 8.14% per year

Total growth 26.4%, but when spread evenly = 8.14% yearly
This accounts for compounding and the migration dip in Year 2-3
```
